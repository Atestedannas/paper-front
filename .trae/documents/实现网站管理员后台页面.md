# 实现网站管理员后台页面

## 1. 需求分析
- 需要独立的管理员登录页面 `/admin/login`
- 需要管理员专属的后台管理页面
- 需要区分普通用户和管理员角色
- 需要管理员专属的功能模块

## 2. 实现方案

### 2.1 路由设计
- 创建独立的管理员路由组，以 `/admin` 为前缀
- 添加管理员路由守卫，检查管理员权限

### 2.2 认证机制
- 在用户信息中添加 `role` 字段，区分普通用户和管理员
- 管理员登录后存储管理员 token 和信息

### 2.3 页面结构
- 创建管理员登录页面 `AdminLoginView.vue`
- 创建管理员后台首页 `AdminDashboardView.vue`
- 创建管理员功能页面（用户管理、论文管理等）
- 创建管理员专属布局组件 `AdminLayout.vue`

### 2.4 权限控制
- 在路由守卫中检查用户角色
- 管理员路由只能由管理员角色访问

## 4. 技术要点

### 4.1 路由设计
```javascript
// 管理员路由组
{
  path: '/admin',
  component: AdminLayout,
  meta: { requiresAuth: true, requiresAdmin: true },
  children: [
    { path: '', redirect: '/admin/dashboard' },
    { path: 'dashboard', name: 'admin-dashboard', component: AdminDashboardView, meta: { title: '管理员控制台' } },
    { path: 'users', name: 'admin-users', component: UserManagementView, meta: { title: '用户管理' } },
    { path: 'papers', name: 'admin-papers', component: PaperManagementView, meta: { title: '论文管理' } },
    { path: 'stats', name: 'admin-stats', component: SystemStatsView, meta: { title: '系统统计' } }
  ]
},
// 管理员登录页
{
  path: '/admin/login',
  name: 'admin-login',
  component: AdminLoginView,
  meta: { title: '管理员登录' }
}
```

### 4.2 路由守卫修改
```javascript
// 检查路由是否需要管理员权限
if (to.matched.some(record => record.meta.requiresAdmin)) {
  const user = getCurrentUser()
  if (!user || user.role !== 'admin') {
    // 非管理员，重定向到普通用户登录页或首页
    next({ path: '/login' })
  } else {
    next()
  }
}
```

### 4.3 管理员登录逻辑
```javascript
// 管理员登录
const adminLogin = async (username, password) => {
  try {
    const response = await request({
      url: '/admin/login',
      method: 'POST',
      data: { username, password }
    })
    // 存储管理员token和信息
    localStorage.setItem('token', response.data.token)
    localStorage.setItem('user', JSON.stringify({ ...response.data.user, role: 'admin' }))
    return response.data
  } catch (error) {
    throw error
  }
}
```
